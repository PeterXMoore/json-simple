The json-simple library is meant to be a lightweight utility for deserializing and serializing things in Javascript Object Notation (JSON). It aims to be easy to learn and use.
json-simple is licensed with a Apache 2.0 license.
Download the json-simple 2.1.1 release jar, javadocs, and source.
As a gradle dependency in a java 7+ project:
runtime 'com.github.cliftonlabs:json-simple:2.1.1'
As a maven dependency in a java 7+ project:
<dependency>
    <groupId>com.github.cliftonlabs</groupId>
    <artifactId>json-simple</artifactId>
    <version>2.1.1</version>
</dependency>
View the repository's change log to see what is new.
Report bugs or request features at the repository's issues page.
Try it out:
/** @param args the command line arguments. 
 * @throws DeserializationException when the example is broken. */
public static void main(String[] args) throws DeserializationException{
    /* Quickly serialize stuff in JSON.simple: */
    Object stuff = null;            
    String serialized = Jsoner.serialize(stuff);

    /* Deserialize stuff in JSON.simple by handling a DeserializationException: */
    Object deserialized = -1;
    try{
        deserialized = Jsoner.deserialize(serialized);
    }catch(DeserializationException caught){
        /* Oops, bad JSON recovery scenario. */
    }

    /* When you know you only want a JsonObject or an empty one if it fails: */
    JsonObject deserializedObject = Jsoner.deserialize(serialized, new JsonObject());

    /* When you know you only want a JsonArray or an empty one if it fails: */
    JsonArray deserializedArray = Jsoner.deserialize(serialized, new JsonArray());

    /* When you want to pretty print into your log: */
    String prettyPrinted = Jsoner.prettyPrint(serialized);

    /* When a specific key is absolutely required: */
    try{
        if(!deserializedObject.containsKey("requiredKey")){
            throw new IllegalStateException("The JSON representation of an object is missing a mandatory key: 'requiredKey'.");
        }
    }catch(IllegalStateException caught){
        /* Oops, bad object recovery scenario. */
    }

    /* When you want a value but have a reasonable default or error value to use if it isn't there: */
    String couldBeTheDefault = deserializedObject.getStringOrDefault("missingKey", "");
    byte couldBeTheErrorValue = deserializedObject.getByteOrDefault("missingKey", (byte)-1);

    /* When you want a value cast to a type in one call but the key is missing: */
    Integer missingValue;
    try{
        missingValue = deserializedObject.getInteger("missingKey");
    }catch(NullPointerException caught){
        /* The cast failed due to the value we expected not being there, indicating we might not be working with data we're expecting.
         * This should have been prevented by checking if the missingKey was there in the first place. */
        missingValue = null;
    }
        
    /* When you want a value cast to a type in one call: */
    deserializedObject.put("key", "1");
    Integer value = deserializedObject.getInteger("key");

    /* Display the output of the example code: */
    System.out.println(stuff);
    System.out.println(serialized);
    System.out.println(deserialized);
    System.out.println(deserializedObject);
    System.out.println(deserializedArray);
    System.out.println(prettyPrinted);
    System.out.println(couldBeTheDefault);
    System.out.println(couldBeTheErrorValue);
    System.out.println(missingValue);
    System.out.println(value);
}

Define how your objects are serialized as JSON by implementing the Jsonable interface.

JsonArray and JsonObject extend ArrayList<Object> and HashMap<String, Object> respectively and have typed getters for each kind of JSON value.

Browse the JSON.simple 2.1.1 release javadocs for more details.
I hope you like where the project is headed and would love to have your support. I've included the project history below for those confused or interested in the origins of the project.

This project was formerly JSON.simple from a Google code project by Yidong.

The 1.* versions of the library had the potential to produce bad JSON, its documentation was uncentralized privately hosted in various locations around the Internet, and with many issues open on the project it seemed the upstream was dead and fairly important to many projects since it had a unique architecture. A few of those projects were internal for Clifton Labs, so it was decided there was value in updating the library and attempt to merge upstream to share our work with anyone else that could benefit.

After redesigning the library to be backwards compatible it was decided that support for java 2 to 6 would be dropped in favor of generics (thus a 2.* version) as it was the most avidly requested feature. The library was also written in the package org.json.simple so a maven artifact under the org.json.simple groupId made sense. Once all that work was done it was up to the merge with upstream to take over the rest. However, after many emails and external issues only the first part went according to plan. So, the project is now maintained by Clifton Labs and the maven artifacts are now published under com.github.cliftonlabs.
